## Sprint 4 概述

### Sprint 目标

**主要目标**: 完成增强功能，为用户测试做好准备

**成功标准**:
- ✅ 完成P0功能（情报简报UI、用户测试框架）

### Sprint 范围

**包含**:
- 情报简报前端UI
- 基础搜索功能
- Markdown导出功能

**不包含**:

## 功能优先级矩阵

### 优先级定义

- **P0**: Must-have - Sprint 4必须完成
- **P1**: Should-have - Sprint 4尽量完成
- **P2**: Nice-to-have - 可推迟到未来Sprint

### 优先级决策矩阵

| 功能 | 用户价值 | 技术复杂度 | 开发成本 | 依赖项 | 优先级 | 理由 |
|------|---------|-----------|---------|-------|-------|------|
| 情报简报UI | 高 | 低 | 低 | 后端已完成 | **P0** | 快速胜利，高影响，后端已就绪 |
| 用户测试 | 极高 | 低 | 低 | 无 | **P0** | 验证产品方向，指导未来开发 |
| 基础搜索 | 高 | 中 | 中 | 无 | **P1** | 显著改善可用性，但非阻断性 |
| Markdown导出 | 中-高 | 低-中 | 低 | 无 | **P1** | 研究者刚需，相对简单 |
| 高级过滤 | 中 | 中 | 中 | 基础搜索 | P2 | 优化搜索体验，可后续添加 |
| PDF报告导出 | 中 | 中 | 中 | 无 | P2 | Markdown导出已满足基本需求 |
| 知识图谱 | 高 | 高 | 高 | 多个AI分析 | P2 | 技术复杂，需要Sprint 5 |

### 优先级总结

**P0 (2项)**:
1. 情报简报UI (Intelligence Briefing Frontend)
2. 用户测试 (User Testing Framework)

**P1 (2项)**:
3. 基础搜索 (Basic Search)
4. Markdown导出 (Markdown Export)

**P2 (3项)**:
5. 高级过滤 (Advanced Filters)
6. PDF报告导出 (PDF Report Export)

## 功能 #1: 情报简报UI (Intelligence Briefing Frontend)

**优先级**: P0
**状态**: 后端60%完成，前端0%
**用户价值**: ⭐⭐⭐⭐⭐
**技术复杂度**: ⭐⭐

### 功能描述

为已完成的Intelligence Briefing后端服务创建用户界面，让用户能够一键生成和查看完整的论文情报简报。

### 用户故事

**User Story 1.1: 生成情报简报**
> 作为研究者，我希望能够一键生成论文的情报简报，以便快速了解论文的核心内容而不必阅读全文。

**验收标准**:
- [ ] 在论文详情页有"生成情报简报"按钮
- [ ] 点击后显示进度指示器（整合情报、生成卡片等阶段）
- [ ] 生成完成后自动跳转到简报视图
- [ ] 生成时间<15秒（后端已实现）
- [ ] 失败时显示友好的错误消息

**User Story 1.2: 查看情报简报**
> 作为研究者，我希望情报简报以清晰、结构化的方式展示，以便我能够快速找到感兴趣的信息。

**验收标准**:
- [ ] 显示案件档案信息（案件编号、标题、完整性评分）
- [ ] 显示Clip 3句摘要
- [ ] 显示结构化信息（研究问题、方法、发现、局限性）
- [ ] 显示AI侦探卡片（按类型分组）
- [ ] 显示用户高亮统计
- [ ] 支持折叠/展开各部分
- [ ] 支持点击卡片跳转到PDF位置

**User Story 1.3: 导出情报简报**
> 作为研究者，我希望能够将情报简报导出为Markdown，以便在论文写作中引用。

**验收标准**:
- [ ] 提供"导出Markdown"按钮
- [ ] 点击后下载.md文件
- [ ] 文件名包含论文标题和日期
- [ ] Markdown格式符合学术规范

### 技术需求

**前端组件**:
```
components/IntelligenceBrief/
├── IntelligenceBriefButton.tsx      # 生成按钮
├── IntelligenceBriefView.tsx        # 简报主视图
├── CaseFileInfo.tsx                 # 案件档案组件
├── ClipSummarySection.tsx           # Clip摘要组件
├── StructuredInfoSection.tsx        # 结构化信息组件
├── ClueCardsSection.tsx             # AI卡片组件
├── UserHighlightsSection.tsx        # 用户高亮组件
├── BriefExportButton.tsx            # 导出按钮
└── IntelligenceBriefProgress.tsx    # 生成进度组件
```

**API集成**:
- 使用现有API: `POST /api/ai/intelligence-brief`
- 请求参数: `{ paperId, pdfText, highlights }`
- 响应类型: `IntelligenceBrief`

**状态管理**:
- 使用Zustand管理简报状态
- 缓存已生成的简报（7天TTL）

**UI/UX要求**:
- 侦探主题设计（报纸风格）
- 响应式布局
- 加载动画（侦探推理动画）
- 错误提示（Toast消息）

### 实施范围

**Sprint 4包含**:
- ✅ 基础UI组件
- ✅ 案件档案展示
- ✅ 所有信息区块展示
- ✅ Markdown导出
- ✅ 折叠/展开功能

**Sprint 4不包含**:
- ❌ 简报编辑功能
- ❌ 简报分享功能
- ❌ 多论文对比分析
- ❌ 知识图谱可视化

### 成本估算

**开发时间**: 2-3天
- UI组件开发: 1.5天
- API集成: 0.5天
- 测试和调试: 0.5天

**AI成本**:
- 已包含在后端服务
- 每篇论文约$0.006-0.010（已验证）

### 风险和依赖

**风险**:
- 后端API可能有bug（需要在集成测试中发现）
- UI可能需要多次迭代

**依赖**:
- 后端`intelligenceBriefService`已实现
- AI API配置正常

## 功能 #3: 基础搜索功能 (Basic Search)

**优先级**: P1
**状态**: 0%完成
**用户价值**: ⭐⭐⭐⭐
**技术复杂度**: ⭐⭐⭐

### 功能描述

实现全文搜索功能，让用户能够快速找到历史笔记、高亮和AI卡片。

### 用户故事

**User Story 3.1: 搜索笔记和卡片**
> 作为研究者，我希望能够搜索关键词，快速找到我之前标记的高亮和生成的AI卡片。

**验收标准**:
- [ ] 在主导航栏有搜索框
- [ ] 支持实时搜索（输入即搜索）
- [ ] 搜索范围：高亮文本、AI卡片标题和内容
- [ ] 显示搜索结果数量
- [ ] 结果高亮显示匹配关键词
- [ ] 点击结果跳转到对应位置

**User Story 3.2: 搜索结果排序和过滤**
> 作为研究者，我希望搜索结果能够按相关度或时间排序，以便快速找到最相关的内容。

**验收标准**:
- [ ] 支持按相关度排序（默认）
- [ ] 支持按时间排序（最新/最旧）
- [ ] 支持按类型过滤（高亮/卡片）
- [ ] 支持按论文过滤

**User Story 3.3: 搜索历史**
> 作为研究者，我希望看到我的搜索历史，以便快速重复之前的搜索。

**验收标准**:
- [ ] 保存最近10条搜索历史
- [ ] 点击历史记录快速填充搜索框
- [ ] 支持清除搜索历史

### 技术需求

**搜索组件**:
```
components/Search/
├── SearchBar.tsx                    # 搜索输入框
├── SearchResults.tsx                # 搜索结果列表
├── SearchResultItem.tsx             # 单个结果项
├── SearchFilters.tsx                # 过滤器
├── SearchHistory.tsx                # 搜索历史
└── useSearch.ts                     # 搜索Hook
```

**搜索算法**:
- 使用Lunr.js或FlexSearch进行全文索引
- 支持中文分词（使用jieba-js或类似）
- 模糊匹配（容错）

**性能优化**:
- 防抖输入（300ms）
- 虚拟滚动（大量结果时）
- 索引缓存

### 实施范围

**Sprint 4包含**:
- ✅ 基础全文搜索
- ✅ 结果高亮
- ✅ 简单排序（时间）
- ✅ 搜索历史

**Sprint 5包含**:
- ❌ 高级过滤（标签、优先级）
- ❌ 语义搜索（需要向量数据库）
- ❌ 搜索建议

### 成本估算

**开发时间**: 2-3天
- 搜索算法实现: 1天
- UI组件开发: 1天
- 集成和测试: 0.5-1天
