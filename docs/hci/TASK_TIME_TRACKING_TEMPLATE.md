# Task Time Tracking Template

**Purpose**: Track task completion times, success rates, and qualitative observations for each participant
**For**: Paper Detective Sprint 4 User Testing (Day 21-22)
**Participants**: 5
**Tasks**: 6

---

## 📋 Overview

### What to Track

For each participant and each task, record:
- ✅ **Completion time**: How long the task took (seconds/minutes)
- ✅ **Success status**: Completed, Abandoned, Failed, or Skipped
- ✅ **Error count**: Number of errors or confusion points
- ✅ **Navigation path**: Clicks, scrolls, page visits
- ✅ **Verbal protocol**: Key quotes from think-aloud
- ✅ **Frustration level**: Low, Medium, High (based on observation)

### Why Time Tracking Matters

**Performance Metrics**:
- Identify which tasks are most difficult (longest time)
- Compare to industry benchmarks
- Track efficiency improvements over time

**UX Insights**:
- Long time = complexity or confusion
- Short time = intuitiveness or skipping
- Abandoned tasks = blockers or bugs

## 📝 Qualitative Observation Codes

### Common Issue Codes

Use these codes to quickly mark common issues during testing:

**Navigation Issues**:
- NAV-CF - Confused about where to click
- NAV-MISS - Couldn't find button/link
- NAV-BACK - Had to go back and try again
- NAV-LOST - Lost, didn't know where they were

**Understanding Issues**:
- UND-TERM - Didn't understand terminology
- UND-LABEL - Label unclear/confusing
- UND-ICON - Icon meaning unclear
- UND-FLOW - Didn't understand workflow

**Technical Issues**:
- TECH-BUG - Bug/crash/error
- TECH-SLOW - Too slow loading
- TECH-FAIL - Feature didn't work
- TECH-ERR - Error message confusing

**Task-Specific Issues**:
- T1-UPLOAD - PDF upload issues
- T2-GEN - Brief generation issues
- T3-SEARCH - Search confusion
- T4-EXPORT - Export problems
- T5-ORG - Organization difficulty
- T6-NAV - Navigation to source failed

### Example Usage

**Task 3 (Search)**:
- NAV-MISS (couldn't find search button)
- UND-FILTER (didn't understand filter options)
- TECH-SLOW (search took too long)

**Task 5 (Organize)**:
- T5-ORG (drag-and-drop confusing)
- NAV-BACK (had to re-read instructions)
- TECH-FAIL (group creation didn't work first time)

## 🎯 Facilitator Quick Reference

### During the Session

**Timing**:
- Start timer when participant begins task
- Stop timer when participant says "done" or moves to next task
- Note if they ask for help (record as hint)
- Note if facilitator intervenes (record as intervention)

**Status Codes**:
- **CS** = Completed Successfully (no help needed)
- **CE** = Completed with Errors (made mistakes but finished)
- **AB** = Abandoned (gave up after trying)
- **FA** = Failed (technical blocker)
- **SK** = Skipped (ran out of time or facilitator skipped)

**Frustration Indicators**:
- **Low**: Calm voice, smooth progression, positive comments
- **Medium**: Sighs, pauses, "hmm", some uncertainty
- **High**: Frustrated tone, multiple errors, negative comments, wants to quit

**What to Record**:
- ALWAYS: Time, status, error count
- IMPORTANT: Verbal quotes (especially "I don't see...", "I expected...")
- HELPFUL: Navigation path (key clicks only)
- OPTIONAL: Exact click sequence (if video recording)

## 📊 Time Analysis Formulas

### Converting Time for Analysis

**Convert to seconds**:
```
Total Seconds = (Minutes × 60) + Seconds
Example: 3:45 = (3 × 60) + 45 = 225 seconds
```

**Average Time**:
```
Average = Sum of all times / Number of participants
Example: (180 + 225 + 195 + 210 + 240) / 5 = 210 seconds = 3:30
```

**Percent Completed**:
```
% Completed = (Completed / Attempted) × 100
Example: 4 out of 5 completed = 80%
```

**Target Success Rate**:
```
Success = If Average Time ≤ Target AND Completion ≥ 80%
Example: Target 3 min, Actual 2:45, Completion 100% = SUCCESS
```

## 🚨 Critical Issue Flagging

### When to Flag Immediately

During testing, flag a task as **CRITICAL ISSUE** if:
- ❌ More than 3 participants (60%+) fail the task
- ❌ More than 3 participants abandon the task
- ❌ Average time is 2× the target
- ❌ High frustration (3/5 or more participants = High frustration)
- ❌ Technical bug blocks completion

**Example Critical Flag**:
```
🚨 CRITICAL ISSUE: Task 3 - Advanced Search
- 3 out of 5 participants failed (60% failure rate)
- Average time: 7:45 (target: <4 min, 194% over target)
- 4 out of 5: High frustration
- Issue: Participants couldn't find filter options

IMMEDIATE ACTION: Fix filter visibility before Day 23
```

## 📋 Session Checklist

### Before Testing
- [ ] Print 5 copies of tracking sheet
- [ ] Test stopwatch/timer
- [ ] Prepare observation codes reference
- [ ] Set up video/screen recording (if using)
- [ ] Review task scenarios one more time

### During Testing
- [ ] Start timer exactly when task begins
- [ ] Record exact times (don't round)
- [ ] Note key quotes verbatim
- [ ] Mark frustration level immediately after task
- [ ] Don't forget to record errors

### After Testing
- [ ] Calculate total session time
- [ ] Verify all 6 tasks have data
- [ ] Transfer data to summary sheet
- [ ] Note any unusual patterns
- [ ] Flag critical issues immediately
