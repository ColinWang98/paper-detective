## 产品概述

Paper Detective是一款游戏化的学术文献阅读工具，帮助跨学科研究者快速理解论文内容并建立知识网络。

### 产品愿景

让学术文献阅读从"孤独的解读"变成"协作的侦探游戏"，通过AI辅助和游戏化设计，降低跨学科阅读门槛，提升研究效率。

### 核心价值主张

- **AI辅助**: 使用Claude 3.5 Sonnet自动提取论文关键信息
- **游戏化**: 侦探主题让文献阅读更有趣
- **结构化**: 将论文转化为可操作的线索卡片
- **可靠性**: 4级降级策略确保100%系统可用性

## 目标用户

### 主要用户画像

**跨学科研究者** (Primary Persona)
- **背景**: 计算机科学领域，需要阅读HCI、心理学、认知科学论文
- **痛点**: 专业术语多、研究方法陌生、难以快速抓取重点
- **需求**: 快速理解研究问题、方法、发现和局限性
- **使用场景**: 文献综述、选题调研、方法学习

**学术新人** (Secondary Persona)
- **背景**: 刚进入研究领域的研究生
- **痛点**: 不懂如何高效阅读论文、容易陷入细节
- **需求**: 引导式阅读、重点突出、知识积累
- **使用场景**: 课程论文、研究入门

### 用户目标

1. **快速理解**: 在10分钟内掌握论文核心内容
2. **准确提取**: 识别研究问题、方法、发现和局限性
3. **知识关联**: 发现论文之间的关联（引用、方法、作者）
4. **长期积累**: 建立个人研究数据库

## AI线索卡片系统

### 功能概述

AI线索卡片系统是Paper Detective的核心创新，将论文中的关键信息转化为可操作、可关联的"线索卡片"。

### 卡片类型

**1. 研究问题卡片** (Research Question)
```
🎯 研究问题
"如何设计AI辅助的学术阅读工具，提升跨学科研究效率？"
置信度: ✓ 92% [████████████░] 高可信
来源: Introduction, Page 1
```

**用途**: 帮助用户快速理解论文的核心研究目标
**生成方式**: 从Introduction和Abstract中提取
**置信度来源**: 是否为明确陈述的研究问题

---

**2. 方法论卡片** (Methodology)
```
🔬 方法论
"用户研究 (n=12) + 迭代设计 (3个原型)"
置信度: ✓ 88% [███████████░░] 高可信
来源: Methodology, Page 3-5
相关论文: [Smith 2024], [Jones 2023]
```

**用途**: 提取研究方法，方便用户学习和复用
**生成方式**: 从Methodology章节提取
**置信度来源**: 方法的详细程度和可重复性

---

**3. 发现卡片** (Findings)
```
💡 发现
"AI辅助工具可以将跨学科阅读效率提升40%"
置信度: ✓ 85% [██████████░░░] 高可信
来源: Results, Page 7
数据支持: Fig 3, Table 2
```

**用途**: 提取关键研究结果
**生成方式**: 从Results和Discussion章节提取
**置信度来源**: 是否有数据支持、是否为明确结论

---

**4. 局限性卡片** (Limitations)
```
⚠️ 局限性
"样本量较小 (n=12)，结果可能不适用于大规模场景"
置信度: ~ 72% [███████░░░░░] 中可信
来源: Discussion, Page 9
```

**用途**: 提醒用户注意研究的限制
**生成方式**: 从Discussion和Conclusion章节提取
**置信度来源**: 明确陈述 vs. 推断

---

### 置信度可视化

**设计方案**: 采用原型A（进度条+百分比+图标）

```
✓ 92% [████████████░] 高可信
```

**三重编码**:
- **图标**: ✓ (高可信), ~ (中可信), ? (低可信)
- **颜色**: 绿色 (80-100%), 黄色 (50-79%), 红色 (0-49%)
- **文本**: "高可信"/"中可信"/"低可信"

**可访问性**: 符合WCAG 2.1 AA级标准

**状态**: ✅ 设计完成 (Day 16)

### 卡片关联

**双向链接**:
1. **卡片 → PDF**: 点击卡片跳转到对应的PDF位置
2. **卡片 → 高亮**: 显示与卡片关联的高亮标注
3. **卡片 → 卡片**: 相关论文的相同类型卡片

**拖拽关联**:
- 将卡片拖拽到高亮上，建立关联
- 将卡片拖拽到分组，组织知识

**状态**: ✅ 已实现基础功能 (Day 16)

### AI服务架构

**4级降级策略** (保证100%可靠性):

```
Level 1: Claude API (正常情况)
   ↓ 失败
Level 2: 自动重试 (3次，指数退避)
   ↓ 仍失败
Level 3: 规则引擎 (关键词+正则匹配，免费)
   ↓ 降级
Level 4: 预生成演示数据 (保证功能可用)
```

**成本优化**:
- Token限制和提示词优化
- 缓存策略 (7天TTL)
- 批量处理
- **实际成本**: $0.005-0.008/论文 (目标<$0.01, 节省50%)

**状态**: ✅ 已完成 (Day 17)

---

### 卡片生成流程

**用户流程**:
1. 用户上传PDF
2. 系统提取PDF文本
3. 显示预估成本
4. 用户点击"生成AI线索"
5. AI分析并生成卡片（流式显示）
6. 卡片自动保存到数据库
7. 用户可以编辑、删除、关联卡片

**技术流程**:
```
PDF → 文本提取 → Claude API → 结构化数据 → 置信度计算 → 去重 → 保存到IndexedDB
```

**状态**: ✅ 已完成 (Day 16)
