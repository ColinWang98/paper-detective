# AI Clue Card Service - Technical Documentation

## Overview
The AI Clue Card Service is responsible for generating and managing AI-powered clue cards that help researchers understand academic papers. This document describes the enhanced implementation with improved confidence scoring, deduplication, and cost optimization.

## Architecture

### Data Model
```typescript
interface AIClueCard {
  id?: number;
  paperId: number;
  type: ClueCardType;        // question | method | finding | limitation
  source: ClueCardSource;    // clip-summary | structured-info | custom-insight | ai-generated
  title: string;             // ≤30 characters
  content: string;           // 1-3 sentences
  pageNumber?: number;
  highlightIds?: number[];
  confidence: number;        // 0-100
  position?: number;
  isExpanded?: boolean;
  createdAt: string;
  updatedAt?: string;
  tokenUsage?: TokenUsage;
  cost?: number;
}
```

### Card Types
1. **🔴 Question** (`question`): Research questions/hypotheses
2. **🔵 Method** (`method`): Methodology and experimental design
3. **🟢 Finding** (`finding`): Key findings and experimental results
4. **🟡 Limitation** (`limitation`): Limitations and unsolved problems

### Card Sources
1. **Clip Summary** (`clip-summary`): Generated from 3-sentence summary
2. **Structured Info** (`structured-info`): Extracted from structured information
3. **Custom Insight** (`custom-insight`): User-generated insights
4. **AI Generated** (`ai-generated`): Directly generated by AI

## Enhancements

### 1. Improved Confidence Scoring

#### Algorithm
The enhanced confidence scoring algorithm combines multiple factors:

```typescript
confidence = (
  informationCompleteness * 0.30 +
  textClarity * 0.25 +
  sourceReliability * 0.20 +
  highlightCorrelation * 0.25
) * 100
```

#### Factors

**Information Completeness** (30% weight)
- Has title (+15%)
- Title length appropriate (10-30 chars) (+10%)
- Has content (+15%)
- Content length appropriate (30-200 chars) (+10%)

**Text Clarity** (25% weight)
- Proper sentence structure (+20%)
- Contains technical terms (+15%)
- Contains quantitative data (+15%)

**Source Reliability** (20% weight)
- structured-info: 95%
- clip-summary: 85%
- ai-generated: 80%
- custom-insight: 70%

**Highlight Correlation** (25% weight)
- Jaccard similarity with user highlights
- Amplified signal (×2) for better correlation

### 2. Card Deduplication

#### Similarity Calculation
```typescript
similarity = titleSimilarity * 0.4 + contentSimilarity * 0.6
```

#### Algorithm
1. Calculate pairwise similarity between cards of the same type
2. If similarity ≥ 0.85 (threshold), mark for deduplication
3. Keep the card with higher confidence score

#### Jaccard Index
```typescript
J(A,B) = |A ∩ B| / |A ∪ B|
```

### 3. Cost Optimization

#### Target
- **Goal**: <$0.01 per paper
- **Model**: Claude Sonnet 4.5
- **Pricing**: $3.0/M input tokens, $15.0/M output tokens

#### Strategies
1. **Input Limiting**: Max 8000 characters from PDF
2. **Highlight Selection**: Top 15 highlights only
3. **Token Estimation**: Accurate pre-calculation
4. **Caching**: 7-day cache for all results

#### Cost Calculation
```typescript
inputCost = (inputTokens / 1,000,000) * $3.0
outputCost = (outputTokens / 1,000,000) * $15.0
totalCost = inputCost + outputCost
```

### 4. Batch Operations

#### Database Optimization
- Batch insert for multiple cards
- Transaction support for data integrity
- IndexedDB indexing for fast queries

#### Performance
- Single API call for multiple cards
- Parallel processing where possible
- Progress callbacks for UI updates

## API Reference

### `generateClueCards(options)`

Generate AI clue cards for a paper.

**Parameters:**
```typescript
interface GenerateClueCardsOptions {
  paperId: number;
  pdfText: string;
  highlights: Highlight[];
  cardTypes?: ClueCardType[];
  onProgress?: (stage: string, progress: number) => void;
  onCardGenerated?: (card: AIClueCard) => void;
}
```

**Returns:**
```typescript
interface ClueCardsGenerationResult {
  cards: AIClueCard[];
  summary: {
    total: number;
    byType: Record<ClueCardType, number>;
    avgConfidence: number;
  };
  tokenUsage: TokenUsage;
  cost: number;
  duration: number;
}
```

### `getClueCards(paperId)`

Retrieve all clue cards for a paper.

### `getClueCardsFiltered(paperId, filter)`

Retrieve filtered clue cards.

**Filter Options:**
```typescript
interface ClueCardFilter {
  types?: ClueCardType[];
  sources?: ClueCardSource[];
  minConfidence?: number;
  pageNumbers?: number[];
  searchQuery?: string;
  dateFrom?: string;
  dateTo?: string;
}
```

### `sortClueCards(cards, sortBy)`

Sort clue cards by specified criteria.

**Sort Options:**
- `created-desc`: Newest first
- `created-asc`: Oldest first
- `confidence-desc`: Highest confidence first
- `confidence-asc`: Lowest confidence first
- `type`: Alphabetical by type
- `page`: By page number

### `updateClueCard(id, changes)`

Update a specific clue card.

### `deleteClueCard(id)`

Delete a specific clue card.

### `deleteClueCardsByPaper(paperId)`

Delete all clue cards for a paper.

### `getClueCardsStats(paperId)`

Get statistics for clue cards.

**Returns:**
```typescript
{
  total: number;
  byType: Record<ClueCardType, number>;
  bySource: Record<ClueCardSource, number>;
  avgConfidence: number;
}
```

## Usage Examples

### Basic Usage
```typescript
import { aiClueCardService } from '@/services/aiClueCardService';

const result = await aiClueCardService.generateClueCards({
  paperId: 1,
  pdfText: paperText,
  highlights: userHighlights,
  onProgress: (stage, progress) => {
    console.log(`${stage}: ${progress}%`);
  },
  onCardGenerated: (card) => {
    console.log('Generated:', card.title);
  }
});

console.log(`Generated ${result.summary.total} cards`);
console.log(`Cost: $${result.cost.toFixed(4)}`);
```

### Filtered Retrieval
```typescript
const cards = await aiClueCardService.getClueCardsFiltered(1, {
  types: ['finding', 'method'],
  minConfidence: 80,
  searchQuery: 'accuracy'
});
```

### Sorting
```typescript
const sorted = aiClueCardService.sortClueCards(
  cards,
  'confidence-desc'
);
```
