## 团队协作流程

### 1. 需求分析阶段
```
Planner + Architect → 技术方案 → UI设计师 → 原型设计
```

### 2. 开发阶段
```
Architect → 接口定义 → Senior Dev-1 + Senior Dev-2 并行开发
                    ↓
            UI设计师 → 组件样式
```

### 3. 审查阶段
```
Code Reviewer → 审查 → 修复 → 合并
```

### 4. 发布阶段
```
Planner → 发布计划 → 团队确认 → 发布
```

## 编码规范

### TypeScript
- 严格模式启用
- 显式返回类型
- 接口优先于类型别名

### React
- 函数组件 + Hooks
- Props接口定义
- 组件文件命名: PascalCase

### CSS
- Tailwind优先
- 自定义CSS在globals.css
- 变量命名: kebab-case

### 测试
- 单元测试覆盖率 > 80%
- 集成测试关键路径
- E2E测试用户流程
